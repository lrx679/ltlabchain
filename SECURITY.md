# LTLab Chain Security Policy

## Reporting a vulnerability

Please report suspected security vulnerabilities privately to the LTLab Chain
maintainers. Do not disclose vulnerabilities in public issues, pull requests, or
social channels before the maintainers have had time to assess and coordinate a
fix.

Include as much detail as possible:

* affected component or command
* software version or commit hash
* reproduction steps
* expected and actual behavior
* potential impact
* logs, traces, or proof-of-concept code when safe to share

## Supported versions

Public support policy and release channels will be documented with LTLab Chain
release artifacts. Operators should run the latest published LTLab Chain release
for their target network unless a specific network notice says otherwise.

## Upstream notices

Some historical audit reports, security references, and vulnerability checks may
still refer to upstream components. LTLab Chain maintainers are responsible for
reviewing whether those notices apply to LTLab Chain releases.
